# Security

If you believe you found a security issue in DingTimer, do not report it in a public issue first.

## Reporting

- Use GitHub's private security advisory flow if it is available for this repository.
- If a private advisory is not available, contact the repository maintainer through the most appropriate private channel you have.
- Include a short summary, reproduction steps, impact, and any relevant logs or screenshots.

## What to Avoid

- Do not publish exploit details publicly before the issue is assessed.
- Do not include secrets, tokens, or personal data in reports unless they are required for reproduction.
